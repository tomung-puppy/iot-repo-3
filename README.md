# 스마트 홈 IoT 통합 프로젝트

현관 RFID 인식부터 엘리베이터 호출, 집 안 조명·커튼·에어컨 제어까지 하나의 흐름으로 자동화하는 스마트 홈 IoT 팀 프로젝트입니다.

## 전체 시나리오

1. **현관 출입 (RFID)**  
   - 사용자가 현관 리더기에 RFID 카드를 태그합니다.  
   - 출입 정보가 서버/DB에 기록되고, 동시에 엘리베이터 호출 이벤트를 발생시킵니다.

2. **엘리베이터 자동 호출**  
   - 서버는 해당 동·호수 정보를 바탕으로 엘리베이터 컨트롤러에 “해당 층으로 이동” 명령을 보냅니다.  
   - 엘리베이터가 사용자가 있는 층으로 자동 이동하여 도착합니다.

3. **실내 입실 & 커튼 자동 제어 (조도 센서)**  
   - 사용자가 집에 들어오면, 실내 조도 센서(LDR)가 현재 밝기를 측정합니다.  
   - 조도가 낮으면 커튼을 자동으로 열어 자연광을 들이고,  
     조도가 너무 밝으면 커튼을 닫아 실내 밝기를 적절하게 유지합니다.

4. **에어컨 시스템 자동 제어 (온도·습도 센서)**  
   - 실내 온도/습도 센서가 현재 환경을 측정합니다.
   - 설정된 쾌적 범위를 벗어나면 에어컨(냉방/제습 등)이 자동으로 ON/OFF 됩니다.
   - 제어 상태(ON/OFF, 목표 온도 등)는 서버/DB에 기록되어 모니터링할 수 있습니다.

## 구성 요소 개요

- **RFID 출입 모듈**
  - 현관에 설치된 RFID 리더기 + 제어 보드
  - 태그 ID, 시간, 사용자 정보 등을 서버로 전송

- **엘리베이터 컨트롤러**
  - 빌딩 엘리베이터 제어용 마이크로컨트롤러 또는 시뮬레이터
  - 호출 층, 현재 층, 이동 방향, 도착 이벤트 등을 관리

- **스마트 커튼 디바이스**
  - 28BYJ-48 스텝모터 + ULN2003 + 조도 센서(LDR)
  - 밝기에 따라 커튼을 자동 개폐하고, 상태/로그를 DB에 기록
  - 상세 내용: `devices/curtain/` 디렉터리 참고

- **온도·습도 기반 에어컨 제어 디바이스**
  - 온도·습도 센서(DHT 계열 등) + 릴레이/적외선 송신 모듈
  - 실내 환경을 기준으로 에어컨 동작 상태를 자동 제어

- **공통 서버 / DB (예: AWS RDS MySQL)**
  - 출입 로그, 엘리베이터 상태, 커튼 로그, 온습도/에어컨 로그 저장
  - 각 디바이스는 공통 포맷으로 로그를 전송하여 하나의 대시보드에서 모니터링 가능

## 디렉터리 구조 (예시)

- `devices/`
  - `curtain/` : 스마트 커튼 디바이스 (아두이노 코드, 업로더, 뷰어, 문서)
  - `rfid/` : 현관 RFID 출입 디바이스
  - `elevator/` : 엘리베이터 제어 디바이스
  - `aircon/` : 온도·습도 기반 에어컨 제어 디바이스
- `docs/`
  - 전체 시스템 아키텍처, ERD, 시퀀스 다이어그램 등
- `scripts/`
  - 공통 유틸 스크립트, DB 초기화 스크립트 등

## 개발 및 실행 개요

각 디바이스별 상세 README는 해당 디렉터리를 참고합니다.  
예) 스마트 커튼: `devices/curtain/docs/README.md`
